# PersonalTeachers v2 - 需求规格说明

> 版本：v2.0 | 日期：2026-04-07

## 1. 项目定位

**一句话**：面向500人的「每日人生智慧」邮件推送平台，用户自定义主题和导师，AI生成个性化语录。

**目标**：
- 秋招简历核心项目，展示 AI 应用开发全栈能力
- 部署在阿里云 2核2G ECS，验证资源受限下的工程能力
- 小规模开放（预期500用户），架构留扩展空间

## 2. 技术栈

| 层 | 技术选型 | 说明 |
|----|---------|------|
| 后端 | FastAPI (Python 3.12+) | AI应用开发主流，异步高性能 |
| 前端 | Vue 3 + Vite + TailwindCSS | 前后端分离，响应式设计 |
| 数据库 | SQLite | 500用户规模足够，零运维，省内存 |
| 定时任务 | APScheduler | 轻量级进程内调度 |
| 邮件 | aiosmtplib | 异步批量发送 |
| AI模型 | OpenAI兼容接口 | DeepSeek + 智谱，统一适配器 |
| 部署 | 阿里云 ECS 2核2G | Nginx 反向代理 |

## 3. 用户系统

### 3.1 注册
- 邮箱注册，发送验证码到邮箱
- 验证码有效期 5 分钟
- 验证通过后设置密码完成注册
- 密码使用 bcrypt 加密存储，禁止明文

### 3.2 登录
- 邮箱 + 密码登录
- JWT Token 鉴权，设置合理过期时间

### 3.3 密码重置
- 通过邮箱验证码验证身份
- 验证通过后设置新密码

## 4. 核心服务 - 每日语录推送

### 4.1 推送内容生成
- 从预设的人生主题/关键字库中，按用户配置的频率权重选取主题
- 从世界人物库中选取导师
- 调用 AI 大模型生成语录（每条语录 = 某位导师关于某主题的智慧之言）
- 每天为每个用户生成 10 条语录，合并为一封邮件发送
- 每人每天最多收到 1 封推送邮件（正常情况），不超过5封

### 4.2 用户个性化配置
- **推送时间**：用户可自定义每日推送时间（如 8:00、12:00、20:00）
- **主题/关键字**：
  - 从预设库中选择感兴趣的主题/关键字
  - 可自行新增自定义主题/关键字
  - 可调整各主题/关键字的出现频率权重
- **人物/导师**：
  - 从预设人物库中选择喜欢的导师
  - 可自行新增自定义人物

### 4.3 防重复发送
- 每个用户每天最多收到一封推送邮件
- 维护「今日已发送」状态，修改推送时间不影响当天已完成的推送

## 5. 邮件系统

### 5.1 多邮箱池
- 后台可配置多个发件邮箱（邮箱地址 + SMTP密码）
- 轮询发送，避免单邮箱触发每日发送限额
- 支持 QQ邮箱、163邮箱、Gmail 等主流邮箱

### 5.2 发送策略
- 按用户设定的推送时间分批调度
- 异步并发发送，控制并发数避免资源耗尽
- 发送失败自动重试（最多3次）
- 记录发送日志，便于排查

### 5.3 SMTP 限制应对
- 主流邮箱每日限额：QQ/163约200-500封，Gmail约500封
- 多邮箱轮询分散发送量
- 500用户 = 500封/天，2-3个邮箱即可覆盖

## 6. 数据管理

### 6.1 历史记录
- 用户的历史推送记录仅保留最近 30 天
- 定时任务自动清理过期数据
- 节省 SQLite 存储空间

### 6.2 数据表设计（核心）
- `users` - 用户信息（邮箱、密码hash、推送时间等）
- `quotes` - 推送记录（用户ID、内容、发送时间、状态）
- `keywords` - 关键词/主题库（预设 + 用户自定义）
- `mentors` - 导师/人物库（预设 + 用户自定义）
- `user_keyword_prefs` - 用户-关键词偏好及权重
- `user_mentor_prefs` - 用户-导师偏好
- `email_pool` - 发件邮箱池
- `ai_models` - AI模型配置
- `verification_codes` - 验证码记录

## 7. 管理后台（独立）

### 7.1 用户管理
- 用户列表、注册时间、最后活跃时间
- 用户增长趋势统计

### 7.2 AI 模型配置
- 配置 AI 厂商的 Base URL、API Key、模型名称
- 内置 DeepSeek 和智谱两个厂商预设
- 支持添加自定义厂商（OpenAI兼容接口）
- 模型优先级 / 降级策略配置

### 7.3 邮箱池管理
- 添加/删除发件邮箱
- 查看各邮箱发送量 / 剩余额度

### 7.4 数据统计
- 总用户数、活跃用户数
- 每日推送成功率/失败率
- 热门主题/导师排行
- 邮件发送量统计

## 8. 前端设计

### 8.1 用户端
- **响应式设计**：PC + 手机自适应
- 页面：
  - 登录/注册/重置密码
  - 仪表盘（今日语录、快捷操作）
  - 主题/关键字管理（选择、新增、调权重）
  - 导师/人物管理（选择、新增）
  - 推送设置（时间、频率）
  - 历史语录查看

### 8.2 管理后台（独立）
- 页面：
  - 数据看板（用户统计、推送统计）
  - 用户管理
  - AI 模型配置
  - 邮箱池管理
  - 系统设置

## 9. 部署方案

- **服务器**：阿里云 ECS 2核2G
- **反向代理**：Nginx
- **进程管理**：systemd 或 Supervisor
- **SSL**：Let's Encrypt 免费证书
- **域名**：需自备或使用 IP 访问

## 10. 非功能性需求

- **安全性**：密码bcrypt加密、JWT鉴权、验证码机制、防重复发送
- **可靠性**：AI模型降级容错、邮件发送重试、异常日志记录
- **性能**：异步IO、批量邮件并发控制、SQLite定期清理
- **可扩展**：用户规模从500扩展到5000时，只需升级服务器或切换到PostgreSQL
